# Manual do Usuário — Multiplica Evidências Linha 2

## Apresentação

Este manual foi preparado para apoiar o uso da linha avançada do `Multiplica Evidências`.

O foco é oferecer orientação clara, com linguagem simples e aplicação prática no cotidiano do professor multiplicador.

Se você preferir uma leitura mais curta, consulte primeiro `docs/guia_rapido.md`.

## O que é o sistema

O `Multiplica Evidências` é uma aplicação desktop que ajuda a registrar, organizar e revisar informações do trabalho pedagógico.

Com ele, é possível acompanhar:

- dados do professor multiplicador;
- turmas;
- encontros;
- evidências;
- checklist mensal;
- relatórios;
- cursistas, ocorrências e socializações.

## Para quem este manual foi pensado

Este material foi pensado principalmente para:

- professores multiplicadores;
- usuários que preferem instruções passo a passo;
- pessoas que precisam consultar o sistema por tarefa, e não por linguagem técnica.

## Como usar este manual

Use este material como apoio em três situações:

- quando estiver começando a usar o sistema;
- quando surgir dúvida sobre uma tela;
- quando precisar revisar um procedimento antes de salvar ou importar dados.

## Passo 1 — Organize a base do sistema

Antes de registrar encontros ou cursistas, faça o preenchimento inicial:

1. Acesse a aba `Professor Multiplicador`.
2. Confira seus dados.
3. Acesse a aba `Turmas`.
4. Cadastre ou revise as turmas.
5. Selecione o mês correto na parte superior da janela.

### Atenção

Se a turma estiver incorreta, vários registros futuros também poderão ficar incorretos.

## Passo 2 — Registre os encontros

Na aba `Encontros`, registre cada encontro pedagógico com atenção aos campos principais.

Revise especialmente:

- data;
- turma;
- pauta;
- observação;
- evidências relacionadas.

### Resultado esperado

Ao final, o encontro deve ficar disponível para consulta e para uso em checklist e relatórios.

## Passo 3 — Revise checklist e relatório

Depois de registrar os encontros, acesse `Checklist e relatório`.

Nessa etapa, o objetivo é verificar:

- o que já está completo;
- o que ainda precisa de ajuste;
- se há aviso de divergência que mereça conferência antes de salvar.

### Atenção

Os relatórios dependem da qualidade dos dados anteriores. Se o cadastro-base estiver inconsistente, o resultado também ficará inconsistente.

## Passo 4 — Acompanhe os cursistas

Na aba `Acompanhamento de Cursistas`, o sistema organiza o trabalho em três frentes.

### `Cursistas`

Use para:

- cadastrar nome e contatos;
- vincular o cursista à turma;
- registrar observação geral;
- marcar situação de busca ativa;
- indicar se o cursista está ativo.

### `Ocorrências`

Use para registrar e acompanhar situações como:

- cursista em turma errada;
- cursista em curso errado;
- problema de inscrição;
- dificuldade de acesso;
- necessidade de encaminhamento ao PEC.

### `Socializações`

Use para acompanhar:

- se a socialização foi enviada;
- se o cursista precisa de apoio;
- se há potencial de destaque;
- observações pedagógicas;
- movimentações e anexos.

## Busca ativa

A busca ativa é útil quando o cursista ainda não apareceu, não respondeu ou precisa ser localizado.

Sempre que possível, mantenha registrados:

- e-mail institucional;
- e-mail pessoal;
- telefone ou WhatsApp.

### Boa prática

Registre apenas o necessário para o acompanhamento. Evite guardar informação pessoal sem utilidade prática.

## Movimentações

As movimentações funcionam como histórico resumido.

Exemplos:

- contato realizado por WhatsApp;
- e-mail enviado;
- aguardando retorno;
- encaminhado ao PEC;
- situação resolvida.

### Boa prática

Escreva movimentações curtas, objetivas e com foco no que realmente mudou.

## Anexos

Os anexos ajudam a guardar evidências complementares.

Exemplos:

- print de tela;
- comprovante;
- arquivo recebido;
- registro de socialização.

### Atenção

Anexe apenas o que tiver utilidade real. Isso evita excesso de arquivos e reduz risco de exposição desnecessária de dados.

## Importação de cursistas por planilha

Quando houver planilha enviada pelos PECs, o sistema pode fazer uma análise antes de importar.

Essa análise ajuda a identificar:

- quem é novo;
- quem pode ser atualizado;
- onde há conflito;
- o que deve ser ignorado.

### Regras adotadas nesta linha

- CPF não deve ser armazenado;
- `di` deve ser ignorado;
- `turmaid` deve ser ignorado;
- o número da turma pode ser extraído do texto da coluna de turma;
- observações no nome após hífen podem ser desconsideradas na análise.

## Backup do banco

O backup não é opcional na prática. Ele deve fazer parte da rotina.

### Recomenda-se fazer backup

- antes de importações;
- antes de alterações maiores;
- ao final do mês;
- sempre que houver registro importante novo.

## Dúvidas comuns

### Não encontrei um cursista

Revise:

- mês selecionado;
- turma escolhida;
- filtros ativos;
- status aplicado.

### A turma parece errada

Confirme:

- o cadastro da turma;
- a origem da planilha;
- se o cursista foi vinculado à turma correta.

### A socialização não mudou de status

Confira:

- o cursista carregado;
- a turma;
- a data de envio;
- o status salvo.
